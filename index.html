Explain it like I am 10

Think of this app like a smart office:

  • Router Agent is the front desk deciding where your question should go.
  • GTM Agent handles product and pricing questions.
  • Outreach Agent writes marketing content and can send emails when intent is explicit.

Each step uses the same shared state backpack: AgentState with question, context, answer, and trace steps.

Simple view: Router picks GTM or Outreach, then rule checks decide whether flow continues or ends.

GTM path

  • gtm_retrieve gathers context from knowledge base + web.
  • pricing_gate checks if pricing was requested.
  • collect_email requires verified email for pricing details.
  • gtm_generate returns final product answer.

Outreach path

  • outreach_research uses Apollo for lead requests, else KB + web.
  • outreach_generate drafts content.
  • send_gate decides send vs review.
  • outreach_send sends via SendGrid when requested.
